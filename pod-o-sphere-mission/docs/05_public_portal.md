# 05 Public Portal

## Portal goals

The public portal should feel like the creator's branded knowledge base, not a generic transcript dump.

## Core pages

### Home

- Show logo
- Banner image
- Description
- Search box
- Featured topics
- Recent episodes
- Popular tags

### Search results

- Query box
- Filters for tag/topic/date/guest
- Episode cards
- Thumbnail
- Result excerpt
- Timestamp links
- Relevance score/debug hidden unless admin mode

### Episode detail

- Title
- Thumbnail
- Published date
- Summary
- Key moments/timestamps
- Tags
- Topics
- Transcript viewer
- Related episodes

### Topic detail

- Topic description
- Episodes where topic appears
- Timeline chart
- Related tags
- Representative quotes/excerpts

### Tag detail

- Episodes with tag
- Related topics
- Sort by relevance/date

## Branding controls

Client admin should control:

- Portal display name
- Logo image
- Banner image
- Primary hex color
- Secondary hex color
- Accent hex color
- Background hex color
- Text hex color
- Button style later
- Font pairing later

## SEO notes

Generate crawlable pages for:

- Episode summaries
- Topic pages
- Tag pages

Avoid exposing full transcripts by default until copyright/provider terms are reviewed. Public excerpts and summaries are safer for MVP.
